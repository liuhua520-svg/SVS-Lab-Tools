# SVS 랩 얼라이너

**노래 음성 합성 랩 얼라이너**

노래 음성 합성(SVS) 워크플로우를 위한 완전 독립형 웹 애플리케이션으로, 입력 오디오와 텍스트를 노래 음성 합성 소프트웨어에서 사용할 수 있는 프로젝트 파일로 변환합니다.

이 애플리케이션의 목표는 TTS(텍스트 음성 변환)를 만들거나 음성 합성 엔진 자체를 대체하는 것이 아니라, 가수가 "말을 잘하는 척"할 수 있도록 도와주는 다리 역할을 하는 것입니다.

- 먼저, **[MFA](https://github.com/MontrealCorpusTools/Montreal-Forced-Aligner)**를 사용하여 오디오와 텍스트를 자동으로 정렬하고 `.lab` 파일을 생성합니다.

- 다음으로, **[PyWORLD](https://github.com/JeremyCCHsu/Python-Wrapper-for-World-Vocoder)**를 사용하여 기본 주파수(F0)를 추출합니다.

- 마지막으로, 소프트웨어 내에서 추가 편집 및 재생을 위해 음성 합성 프로젝트 파일을 출력합니다.

## ✨ 주요 기능

* **어노테이션만 (빠름)**
MFA 자동 정렬만 수행하여 `.lab` 파일을 빠르게 생성합니다.

* **완전 처리 (어노테이션 + F0 + 프로젝트 파일)**
어노테이션, 피치 추출을 자동으로 완료하고 보컬 합성 프로젝트 파일을 출력합니다.

* **프로젝트 생성만 (WAV + LAB)**
기존 `WAV + LAB` 파일을 기반으로 프로젝트 파일을 직접 생성하여 어노테이션을 다시 입력할 필요가 없습니다.

* **다양한 피치 추출 알고리즘 지원**
내장된 **PyWORLD, CREPE, RMVPE** 세 가지 F0 추출 방식을 통해 필요에 따라 다양한 알고리즘을 선택할 수 있습니다.

* **연속 피치 기록 지원**
순수 음표만 생성하거나 연속적인 F0 곡선을 ​​프로젝트에 기록하여 보다 사실적인 피치 변화를 유지할 수 있는 옵션을 제공합니다.

* **긴 오디오 파일 처리 지원**
긴 오디오 파일 처리가 가능하여 노래 전체, 긴 음성 시퀀스 등 다양한 상황에 적합합니다.

* **다양한 텍스트 변환 방식 지원**
"프로젝트만 생성(WAV + LAB)" 모드에서 다음 옵션을 선택할 수 있습니다.

* 변환 없음

* 자음 병합

* 히라가나

* 가타카나

* **다양한 오디오 형식 지원**
`WAV / MP3 / FLAC / M4A / AAC`

* **다국어 텍스트 정렬 지원**

* **LAB 파일 내보내기 지원**

* **다양한 정렬 알고리즘 지원**

MFA, WhisperX, Qwen3-ASR, Qwen3-FA 등 네 가지 내장 정렬 방식을 제공하여 사용자가 특정 요구 사항에 따라 적절한 알고리즘을 선택할 수 있습니다.

* **간편하고 사용하기 쉬운 웹 인터페이스 제공**

## 🛠️ 사용 방법

1. 오디오 파일과 텍스트를 입력합니다.

2. **[MFA](https://github.com/MontrealCorpusTools/Montreal-Forced-Aligner)**를 사용하여 자동으로 시간 정렬을 합니다.

3. `.lab` 파일을 생성합니다. 어노테이션 파일

4. **[PyWORLD](https://github.com/JeremyCCHsu/Python-Wrapper-for-World-Vocoder)**를 사용하여 F0 피치를 추출합니다.

5. 처리 모드에 따라 프로젝트 파일을 출력합니다.

6. 결과를 음성 합성 소프트웨어로 가져와 추가 작업에 사용합니다.

## 🔧 처리 모드

### 1. 어노테이션만 (빠른)

자동 MFA 어노테이션만 수행하고 .lab 파일을 생성합니다.

### 2. 전체 처리 (어노테이션 + F0 + 프로젝트 파일)

MFA 정렬, F0 추출을 수행하고 프로젝트 파일을 생성합니다.

### 3. 프로젝트 파일만 생성 (WAV + LAB)

기존 WAV 및 LAB 파일을 사용하여 프로젝트 파일을 생성합니다. 어노테이션 과정은 반복되지 않습니다.

### 📋 시스템 요구 사항

Node.js 16 이상

4GB RAM

1GB 이상의 디스크 여유 공간

pip install montreal-forced-aligner

## 📖 사용 흐름

### 주석만 (빠른 버전)

1. 이 애플리케이션을 엽니다.

2. 오디오 파일을 업로드합니다.

3. 텍스트를 입력합니다.

4. 언어를 선택합니다.

5. 주석 시작 버튼을 클릭합니다.

6. 생성된 .lab 파일을 다운로드합니다.

### 전체 처리 (주석 + F0 + 프로젝트 파일)

1. 오디오 파일을 업로드합니다. 파일

2. 1. 텍스트 입력

2. 언어 선택

3. 처리 모드를 "전체 처리"로 선택

4. 처리 시작 클릭

5. 생성된 프로젝트 파일 다운로드

### 프로젝트만 생성 (WAV + LAB)

1. WAV 파일과 해당 LAB 파일을 준비합니다.

2. "프로젝트만 생성"을 선택합니다.

3. WAV + LAB 파일을 업로드합니다.

4. 생성 클릭

5. 프로젝트 파일 다운로드

### 📝 출력 내용

- .lab 시간 정렬 파일

- F0 피치 정보

- 보컬 합성 프로젝트 파일

### 📌 기능 제한

이 도구는 "SVS 워크플로우 보조 도구"로만 사용되며 다음과 같은 기능은 제공하지 않습니다.

- 음성 합성(TTS / SVS) 기능을 제공하지 않습니다.

- 보이스뱅크 모델을 학습시키거나 수정하지 않습니다.

- 음성 합성 소프트웨어를 대체하지 않습니다.

---

### 📁 사용 책임

이 도구를 사용하여 생성된 결과물(`.lab` 파일, F0 데이터 및 프로젝트 파일 포함)에 대한 책임은 사용자에게 있습니다.

- 사용자는 결과물의 적법성 및 사용 목적에 대한 전적인 책임을 집니다.

- 사용 전에 사용된 오디오 및 보이스뱅크의 라이선스 범위를 확인하십시오.

- 타사 보이스뱅크를 사용하는 경우, 해당 보이스뱅크의 공식 이용 약관을 준수해 주십시오.

---

### 🎭 보이스뱅크 및 주석 관련 유의사항

보이스뱅크 또는 합성 시스템에 저작자 표시가 필요한 경우, 해당 라이선스에 따라 주석을 달아주십시오. (변환된 보이스뱅크 파일의 사운드가 원본 샘플의 특성을 갖는 것으로 확인될 경우, 연기 지침 표시 여부가 결정됩니다. 만약 그렇다면, 해당 라이선스에 따라 주석을 달아주십시오.) 예시:

- "사용된 보이스뱅크: XXX"

- "연기 지침: XXX"

---

### 📌 저작권 고지

본 프로젝트는 사용자 입력 데이터 또는 생성된 콘텐츠에 대한 소유권을 주장하지 않습니다.

데이터 저작권은 사용된 원본 샘플(변환된 보이스뱅크 파일의 사운드)이 원본 샘플의 특성을 갖는 것으로 확인되는지 여부에 따라 사용자에게 있습니다. 타사 원본 샘플이 원본 샘플의 특성을 갖는 것으로 확인될 경우, 해당 타사의 파생 저작물로 간주될 수 있습니다.
